# 项目需求文档

## 目标

构建一个本地 Web 版小红书运营后台，用于品牌账号的内容发布、搜索监控、主页查询和账号分析。当前版本不考虑 Android App 自动化、不考虑私信能力、不考虑向外部数据接口上传。

## 目标用户

- 品牌运营团队
- 内部运营执行人员

## 功能范围

### 账号管理

- 保存多个小红书账号 Cookie。
- 支持 PC 端扫码登录，登录成功后自动保存 Cookie。
- 手动 Cookie 保存时账号名可留空，后端会尝试自动获取昵称。
- 切换当前操作账号。
- 校验 Cookie 可用性。
- 删除账号。
- 前端不展示完整 Cookie。

### 内容发布

- 支持输入标题、正文、地点。
- 支持图文和视频发布。
- 发布表单合并直接发布和第三方二维码发布入口。
- 媒体支持本地上传和链接两种来源。
- 本地上传用于账号 Cookie 直接发布。
- 链接媒体用于第三方二维码发布，要求图片/视频 URL 可公网访问。
- 发布流程中自动从正文里的 `#话题` 提取话题标签。
- 发布前校验账号状态。
- 直接发布和二维码发布成功后都记录到发布历史，便于后续复用。
- 发布历史支持一键带入发布页继续编辑。
- 记录发布历史状态和操作日志。
- 支持第三方二维码发布接口：保存 API Key，提交在线图片/视频 URL，生成手机扫码发布二维码。

### 关键词查询与监控

- 支持按关键词搜索笔记。
- 支持排序、类型、时间范围、数量参数。
- 搜索结果默认只展示摘要。
- 正文、图片、视频、发布时间按需加载。
- 支持保存关键词监控任务。
- 支持手动执行关键词监控。
- 监控结果本地去重保存。

### 主页查询

- 查询当前账号主页。
- 通过主页链接或 user_id 查询别人主页。
- 查询主页笔记分页列表。
- 主页笔记和关键词笔记复用同一套卡片布局。

### 账号分析

- 采集当前账号主页基础指标。
- 采集最近笔记摘要。
- 保存账号快照。

### 风控策略

所有需求必须遵守根目录 [RISK_CONTROL.md](../RISK_CONTROL.md)。

### 国际化

- Web 页面提供中英文切换入口。
- 第一版优先覆盖导航、核心入口和新增二维码发布流程。

### UI 与响应式

- 前端使用 Tailwind CSS 和 shadcn/ui 配置体系。
- 视觉风格按 iOS 浅色分组、圆角卡片、轻阴影和底部移动端导航定制。
- 页面必须适配桌面端和移动端。

## 暂不包含

- 私信监控与回复。
- Android App 自动化。
- 上传数据到外部后端。
- 公网部署和多用户权限系统。
- 验证码绕过、高频并发、批量账号并发。
- 页面内手机号验证码登录。
- 创作者平台扫码登录。
