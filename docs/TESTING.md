# 自动化测试

## 测试目标

第一版测试体系用于保护 Web 运营后台的基础能力，避免后续开发破坏已有需求。

## 覆盖范围

- 后端健康检查。
- 账号新增和 Cookie 脱敏。
- 发布历史创建、列表、删除。
- 关键词监控创建、列表、删除。
- 关键词监控参数边界校验。
- 关键词监控结果按笔记去重。
- 无标题、未知作者且无封面的无效搜索摘要过滤。
- 第三方扫码发布 API Key 保存和脱敏展示。
- 第三方扫码发布参数校验。
- 第三方扫码发布二维码接口 mock 调用。
- 前端 TypeScript 和 Vite 构建。

## 当前测试缺口

- 尚未接入浏览器端自动化测试，暂时不能覆盖表单交互和错误提示渲染。
- 后续需要补 Playwright 冒烟测试，至少覆盖保存监控、发布页、发布历史带入发布。

## 运行方式

后端测试：

```bash
python -m pytest
```

前端构建：

```bash
cd frontend
npm run build
```

一键检查：

```bash
sh scripts/check.sh
```

## 约定

- 测试不能真实请求小红书。
- 测试不能真实请求第三方发布服务。
- 测试不能使用真实 Cookie 或 API Key。
- 新增后端接口时，优先补对应 TestClient 测试。
- 新增前端关键流程时，至少保证 `npm run build` 通过；复杂交互后续补 Vitest 或 Playwright。
